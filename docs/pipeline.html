Pipeline

Aaron's auto-inpainting workflow at the core. Three focus areas wrap around it — avatars, try-on generation, and quality control. Runs locally today; target is AWS batch.

Reference photo (today often from phone — quality risk).

→

Three focus areas

Avatars

Per-brand identity

Brand-specific synthetic model identity. Includes pose library and lighting library so the same avatar can render across angles without drifting.

Setup needs

  • Pose library per brand
  • Lighting library per brand
  • Naming conventions (per client) link garment → avatar slot

Try-on generation

Garment × avatar × angle

Collage technique preserves avatar likeness across multiple angles. Spreadsheet logic drives which avatar × which garment × which pose, queues API calls.

Known limitations

  • Pockets don't render correctly with hand positioning
  • Crop shots stuck at full / upper / lower body — anything tighter needs dev work
  • Color accuracy breaks with varied input lighting

Quality control

Every image, every time

Heidi + Terry review each generated image. Industry background in tech packs informs what they flag. Manual workflow still produces better lighting + quality than automated; automated has consistency issues.
